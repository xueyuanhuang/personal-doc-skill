# Personal Doc Skill — AI 生成个人使用说明书

> 收集素材 → 抓取全网数据 → 并行分析 → 生成说明书 → 迭代修改 → 导出

## Trigger

- `/personal-doc`
- 用户表达想生成个人说明书的意图（"帮我写个人说明书"、"入职要填个人介绍"）

## 概述

基于用户的简历、测评报告、社交媒体全量内容，用 AI 自动分析并生成一份 RockFlow 入职用的「个人使用说明书」。核心思路：**让 AI 从你实际写过的内容中认识你，而不是让你从零描述自己。**

## 流程

每一步输出后等用户确认再进入下一步。

---

### Step 1: 收集基础素材

向用户收集以下素材（用 AskUserQuestion 确认哪些可提供）：

**必填：**
- 简历（PDF/文件路径/直接粘贴）
- 入职岗位和部门

**选填（越多越好）：**
- 测评报告：盖洛普优势识别器、Principles You、MBTI 等
- 微信公众号名称或文章目录路径（如已下载）
- Twitter 用户名
- 小红书 cookie（a1 + web_session）
- 其他内容平台账号

输出格式：
```
**已收集素材：**
- [x] 简历
- [x] 岗位：XX
- [x] 盖洛普测评
- [ ] 微信公众号（待抓取）
- [ ] Twitter（待抓取）
- [ ] 小红书（待抓取）
```

等用户确认后进入下一步。

---

### Step 2: 抓取社交媒体数据

根据 Step 1 确认的数据源，**并行抓取**所有社交媒体数据。每个平台的抓取方法：

#### 微信公众号

如果用户已有本地文章文件（markdown/txt），直接读取。

如果需要下载，让用户提供文章导出方式（微信公众平台后台导出、第三方工具等），协助处理成可分析的格式。

#### Twitter

使用 opentwitter MCP 工具抓取：
```
1. 用 get_twitter_user 获取用户信息
2. 用 get_twitter_user_tweets 分页获取全量推文（每页最多 40 条，用 cursor 翻页）
3. 保存为结构化数据
```

#### 小红书

小红书反爬较强，推荐方案：**Playwright MCP 浏览器拦截法**

```
前提：用户提供小红书 cookie（a1 + web_session）

步骤：
1. 用 Playwright MCP 的 browser_run_code 注入 cookie
2. 用 browser_navigate 访问 https://www.xiaohongshu.com/user/profile/self
3. 确认跳转到用户主页（URL 变为 /user/profile/{user_id}）
4. 用 browser_run_code 注入 API 响应拦截器：
   - 监听 fetch/XHR 响应
   - 匹配 /api/sns/web/v1/user_posted 接口
   - 将每页返回的 notes 数据存入 window.__captured_notes
5. 用 browser_evaluate 滚动页面，触发分页加载
6. 重复滚动直到 has_more=false 或连续 5 次无新数据
7. 用 browser_run_code 提取 window.__captured_notes
```

**注意事项：**
- 不要用 headless 模式，小红书会检测
- 不要用 xhs Python 库直接调 API（签名机制会失败）
- 如果触发验证码，停止抓取，使用已获取的数据继续
- cookie 有效期有限，如果返回登录页说明 cookie 已过期

#### 抓取完成后

输出数据汇总：
```
**数据抓取完成：**
- 微信公众号：XXX 篇文章
- Twitter：XXX 条推文
- 小红书：XXX 条笔记
- 测评报告：X 份
- 简历：1 份

总计 XXX 条内容素材
```

将所有数据保存到工作目录下的 `关于我/` 子目录。

---

### Step 3: 并行分析

启动 **4 个并行分析 Agent**（使用 Task 工具，subagent_type=general-purpose），每个分析不同维度：

**Agent 1 — 性格与行为模式**
```
分析全量内容，提取：
- 性格特征（直接/内敛、执行力/思考型、乐观/谨慎等）
- 行为模式（面对压力、面对失败、面对机会的典型反应）
- 自我认知（作者怎么描述自己、自嘲的点、承认的短板）
- 代表性例子和原话
```

**Agent 2 — 工作风格与能力**
```
分析全量内容，提取：
- 专业技能和工具栈
- 工作方法论（数据驱动/直觉驱动、独立/协作偏好）
- 对效率和自动化的态度
- 过往项目和成果的具体数据
- 代表性例子和原话
```

**Agent 3 — 兴趣、价值观与生活**
```
分析全量内容，提取：
- 兴趣爱好的广度和深度（不只是列表，要有具体参与度的证据）
- 核心价值观（从反复出现的观点中提炼）
- 生活方式和偏好
- 推荐的书/电影/内容及理由
- 代表性例子和原话
```

**Agent 4 — 金句与高频观点**
```
分析全量内容，提取：
- 反复出现的核心观点（出现 3 次以上的主题）
- 最能代表作者的原话/金句（10-15 句）
- 协作偏好相关的表述
- 对团队/组织的期望
```

如果文章数量超过 50 篇，每个 Agent 分批处理（每批 30-50 篇），最后合并结果。

---

### Step 4: 生成说明书

汇总 4 个 Agent 的分析结果 + 简历 + 测评数据，按照 `template.md` 的结构生成完整的个人使用说明书。

**写作原则：**
- 用第一人称，语气自然不端着
- 每个观点都要有来自原始内容的具体例子或数据支撑
- 不要写「正确的废话」——如果一个描述换个名字也成立，那就不够具体
- 性格描述要包含「短板」和「需要团队配合的点」，不要只写优点
- 引用原话时用 blockquote 格式，标注来源平台
- 「在 RockFlow 的工作」部分需要结合岗位需求和个人优势来写

将生成的说明书输出在对话中，等用户审阅。

---

### Step 5: 迭代修改

用户提修改意见 → 调整 → 输出完整修改后全文 → 直到用户满意。

注意：
- 每次修改输出完整全文
- 如果用户说"可以了"/"没问题"，进入下一步

---

### Step 6: 导出

用户确认定稿后：

1. **保存 .md 文件**到工作目录（如 `关于我/个人说明书.md`）
2. **格式说明**：
   - 文档标题不写在 .md 里（飞书导入时单独设置）
   - 所有章节用 `#`（一级标题），不用 `##`
   - 开头保留 AI 生成声明（blockquote 格式）
3. 提示用户：
   ```
   文件已保存到：[路径]

   导入飞书：
   1. 打开飞书文档
   2. 导入 .md 文件
   3. 手动设置文档标题
   4. 上传个人照片到「个人照片」部分
   ```

---

## Rules

1. **严格按步骤对话**：每一步只输出当前步骤的内容，等用户确认再继续
2. 用中文交流
3. 抓取数据时优先并行，最大化效率
4. 分析时 4 个 Agent 必须并行启动
5. 保护用户隐私：cookie、token 等敏感信息不要写入最终文档
6. 如果某个平台抓取失败，用已有数据继续，不要阻塞整个流程
7. 生成的内容要基于真实数据，不要编造例子或数据
8. 小红书抓取最容易出问题——如果失败，优先用其他平台数据生成，小红书数据作为后续补充
